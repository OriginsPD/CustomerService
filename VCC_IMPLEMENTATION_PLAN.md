# VCC — Virtual Customer Care Office Implementation Plan

> Generated: 2026-03-18 | Based on: VCC_Project_Roadmap.pdf

---

## Context

Building a greenfield AI-powered customer service application from the VCC Project Roadmap PDF. The user has selected **Opencode SDK** (`@opencode/sdk`) as the AI framework with a modern TypeScript full-stack monorepo. The repository is currently empty. This is a 2-week proof-of-concept sprint covering four modules: Check-In, Check-Out, Self-Evolving Feedback System, and Analytics Dashboard.

---

## Architecture Decision

**Monorepo** with npm workspaces — three packages run concurrently in dev:

| Package | Tech | Port |
|---|---|---|
| `client/` | Vite + React 19 + TanStack Router v1 + TanStack Query v5 | 5173 |
| `server/` | Hono (TypeScript HTTP framework) + Drizzle ORM | 3001 |
| `packages/shared/` | Zod schemas + TypeScript types (shared contract) | — |

Vite proxies `/api/*` → `localhost:3001` so no CORS issues in dev. ShadCN UI configured for Vite (not Next.js). Dashboard is open (no auth) for PoC. 24h scheduler runs automatically + exposed via `POST /api/admin/run-analysis` for manual testing.

## Database Schema (Drizzle ORM → Neon PostgreSQL)

### Enums
```typescript
sessionStatusEnum: "waiting" | "in_progress" | "completed" | "cancelled"
sentimentEnum:     "positive" | "neutral" | "negative"
questionTypeEnum:  "text" | "boolean" | "scale"
questionSourceEnum:"ai_generated" | "manual"
aiActionEnum:      "add_question" | "remove_question" | "retain"
```

### Tables & Indexes (O(1) access design)

**`sessions`** — Check-In records
- `id` text PK (nanoid), `name`, `email`, `phone`, `company`, `purpose`
- `status`, `queue_number`, `checked_in_at`, `checked_out_at`
- Indexes: `email_idx`, `status_idx`, `status_queue_composite_idx`, `date_idx`

**`feedback`** — Check-Out submissions
- `id` text PK, `session_id` FK → sessions
- `rating` int (1–5), `comment` text, `sentiment`, `sentiment_score` real, `submitted_at`
- Indexes: `session_id_idx` (O(1) FK lookup), `sentiment_idx`, `rating_idx`, `date_idx`

**`feedback_answers`** — Dynamic question responses
- `id` text PK, `feedback_id` FK, `question_id` FK, `answer` text
- Indexes: `feedback_id_idx`, `question_id_idx`, `composite_idx(feedback_id, question_id)`

**`dynamic_questions`** — AI-managed form questions
- `id` text PK, `text`, `type`, `is_active` bool, `source`, `display_order` int
- `added_at`, `removed_at`
- Indexes: `active_idx`, `order_idx`

**`ai_decision_log`** — Full audit trail of AI decisions
- `id` text PK, `run_at`, `action`, `question_id` FK nullable
- `reasoning` text, `feedback_sample_size` int, `raw_ai_response` jsonb
- Indexes: `run_at_idx`, `action_idx`

## O(1) Algorithm Patterns

| Layer | Pattern | Implementation |
|---|---|---|
| DB PK lookups | B-tree index | `sessions.id`, `feedback.id` primary keys |
| FK lookups | B-tree index | `feedback.session_id` indexed |
| Queue position | In-memory Map | `Map<sessionId, queueNumber>` in `queue.service.ts` |
| Active questions | Filtered index | `is_active` indexed, small result set |
| Cache reads | TanStack Query Map | Stable `queryKeys` = O(1) Map lookup, no network |
| Question cache | staleTime 5min | Questions only change every 24h |

**TanStack Query cache keys** (`client/src/lib/queryClient.ts`):
```typescript
export const queryKeys = {
  session:         (id: string)     => ["session", id],
  queuePosition:   (id: string)     => ["queue", "position", id],
  activeQuestions: ()               => ["questions", "active"],
  dashboardSummary:()               => ["dashboard", "summary"],
  trends:          (days: number)   => ["dashboard", "trends", days],
  keywords:        ()               => ["dashboard", "keywords"],
  aiLog:           (page: number)   => ["ai", "log", page],
}
```

## AI Integration (`@opencode/sdk` — `server/src/services/ai.service.ts`)

Single AI boundary. Three exported functions:

1. **`analyzeSentiment(comment)`** — called synchronously on checkout submit
   - Returns: `{ sentiment, score: -1.0 to 1.0, thankYouMessage }`

2. **`generateAdaptiveQuestions(feedbackBatch, existingQuestions)`** — called by 24h scheduler
   - Returns: `{ action: "add"|"remove"|"retain", questionText?, questionId?, reasoning }[]`
   - Keeps max 5 active questions; adds for patterns/gaps; removes for low engagement

3. **`extractKeywords(comments[])`** — called by dashboard keywords endpoint
   - Returns: `{ word, count, sentiment }[]`

**24h Scheduler** (`server/src/services/scheduler.service.ts`):
```
export async function runAnalysis(): Promise<void> {
  1. SELECT last 24h feedback + answers
  2. SELECT active questions
  3. ai.generateAdaptiveQuestions(batch, questions)
  4. DB writes: insert/deactivate questions + ai_decision_log
}

startScheduler(): setInterval(runAnalysis, 24h)  // auto on server start
POST /api/admin/run-analysis → calls runAnalysis() for manual demo triggers
```

## Implementation Order

1. Monorepo scaffold + workspace configs + `.env`
2. `packages/shared` — all Zod schemas (both sides depend on this)
3. `server/src/db/schema.ts` — Drizzle schema → `db:generate` → `db:migrate` on Neon
4. Hono server: checkin → checkout → queue → questions → dashboard → admin routes
5. Vite + React client + TanStack Router + `globals.css` theme
6. Check-In module end-to-end (form → API → Dialog → queue card)
7. Check-Out module (star rating + char-counted comment + dynamic questions)
8. AI service (`analyzeSentiment` first, wired to checkout)
9. 24h scheduler + `generateAdaptiveQuestions` + DB writes
10. Analytics Dashboard (metrics → charts → keyword cloud → AI log Sheet)
11. Polish: animations, gradient borders, gloss effects, responsive layout
